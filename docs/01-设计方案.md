# 设计方案 · 这包豆子怎么冲（CoffeeStory）

> 版本 v1.0　|　阶段：设计方案（Phase 1）
> 文档目标：确立产品定位、核心概念模型、信息架构、功能范围与设计语言，作为后续《功能需求文档》的输入。

---

## 0. 一句话定位

**不是「咖啡日记」，而是「单包豆子的调参实验室」。**
帮一个手冲玩家把"这包豆子第一次太酸 → 第四次终于调出甜感"的收敛过程记录下来、复盘出来、并沉淀成可复用的配方。

口号（Tagline）：**这包豆子，到底怎么冲？**

---

## 1. 产品定位与核心洞察

### 1.1 竞品已经做了什么
- iBrewCoffee / Cupping Notes 等：记录豆子、配方、研磨、粉量水量水温、萃取时间、风味评分、产地处理法烘焙度。
- 它们的共同心智是 **「日记 / 流水账」**：每一杯是一条独立记录，彼此平铺、互不关联。

### 1.2 我们的差异化洞察
咖啡玩家真正的痛点不是"记录喝过什么"，而是：

> **「同一包豆子，我怎么一次比一次冲得更好？最佳参数是什么？喝完之前能不能把它榨干？」**

所以本产品的第一性原理是 **收敛（Convergence）**，不是 **记录（Logging）**：
- 一包豆子 = 一个"实验项目（Project）"。
- 每一次冲煮 = 一次"实验迭代（Iteration）"，带着"上次太酸→这次磨细一点"的**假设与调整**。
- 项目的终点是**找到最佳参数（Dial-in 完成）**，然后这包喝完"毕业"，最佳配方沉淀进**配方库**。

这条主线，是竞品平铺式日记给不了的。

### 1.3 核心价值闭环
```
买一包新豆  →  不知道怎么冲（未解锁）
   ↓ 第1次：照默认冲 → 太酸，记下"下次磨细"
   ↓ 第2次：磨细一档 → 好一点，记下"下次降水温"
   ↓ 第3次：水温降到90℃ → 甜感出来，接近了
   ↓ 第4次：微调 → 标记为「最佳参数」（解锁！）
   ↓ 继续按最佳参数喝，直到克数耗尽
喝完毕业  →  最佳配方归档进「配方库」
   ↓ 下次买同产区/同处理法的豆子
复用历史配方作为起冲参数  →  少走弯路
```

## 2. 目标用户与使用场景

### 2.1 目标用户
- **核心用户**：有手冲设备（手冲壶/磨豆机/电子秤/温度计）的进阶爱好者。会为"研磨度差一档""水温差 2℃"较真。
- **非目标用户**：只喝速溶/挂耳、对参数无感的轻度用户（不做迎合，避免功能稀释）。

### 2.2 关键场景（按频次）
| 频次 | 场景 | 触发的核心功能 |
|---|---|---|
| **每天 1–3 次** | 早上冲一杯，边冲边计时分段注水 | 冲煮计时器、分段注水记录 |
| **每次冲完** | 喝一口，打分，写"下次怎么调" | 风味打分、复盘小结、下次调整建议 |
| 每包开封时 | 录入一包新豆 | 豆子录入 |
| 隔几天 | 看这包调到第几次了、还剩多少克 | 豆子详情 / 实验曲线、克数进度 |
| 调出最佳后 | 截图发朋友圈/给豆友 | 分享卡片 |
| 买新豆前 | 翻历史最佳配方找起点 | 配方库 |

### 2.3 高频打开的设计承诺
- **冷启动 ≤ 2 秒进入"开始冲煮"**：首页主操作就是"冲一杯"，不需要层层进入。
- 冲煮中全程**单手可操作、不熄屏、计时清晰可见**（站在秤和壶前操作）。

## 3. 核心概念模型（数据 / 状态机）

### 3.1 实体（Entities）

#### Bean（豆子 / 一包）
| 字段 | 说明 | 必填 |
|---|---|---|
| 名称 | 豆子名 / 自定义昵称 | ✅ |
| 烘焙商 Roaster | | ✕ |
| 产区 Origin | 国家/产区/庄园（可自由文本 + 常用建议） | ✕ |
| 处理法 Process | 水洗/日晒/蜜处理/厌氧… 枚举+自定义 | ✕ |
| 烘焙度 Roast | 浅/中浅/中/中深/深（5 档） | ✕ |
| 烘焙日期 Roast Date | 用于推算养豆天数 | ✕（强烈建议） |
| 净含量 Bag Weight | 克（默认常见 200/250/340/454） | ✅ |
| 剩余克数 Remaining | 随每次冲煮自动扣减，可手动校准 | 自动 |
| 风味标签 Flavor Tags | 商家标注的风味（莓果/柑橘/坚果…） | ✕ |
| 封面 Photo | 豆袋照片 | ✕ |
| 备注 Notes | | ✕ |
| 状态 Status | 见状态机 | 自动 |

> **派生字段（不存储，实时计算）**：养豆天数 = 今天 − 烘焙日期；消耗进度 = (净含量 − 剩余)/净含量；本包冲煮次数；本包最高分。

#### Brew（一次冲煮 / 一次实验迭代）
| 字段 | 说明 | 必填 |
|---|---|---|
| 所属豆子 | 外键 → Bean | ✅ |
| 冲煮时间戳 | 自动 | 自动 |
| 研磨度 Grind | 数值（磨豆机刻度，自由数值/文本，记住上次） | ✅ |
| 粉量 Dose | 克 | ✅ |
| 水量 Water | 克 | ✅ |
| 粉水比 Ratio | **派生** = 水量 / 粉量（显示 1:N） | 自动 |
| 水温 Temp | ℃ | ✕ |
| 注水分段 Pours | 段列表（见 PourStage） | ✕ |
| 总萃取时间 Total Time | 秒（计时器产出，可手填） | ✕ |
| 风味评分 Flavor | 多维（见下） | ✕ |
| 一句话小结 Takeaway | "太酸""甜感出来了" | ✕ |
| 下次怎么调 Next Tweak | 结构化（维度+方向）或自由文本 | ✕ |
| 是否最佳 isBest | 每包至多 1 个（标新会取消旧的） | 自动/手动 |
| 评分总分 Overall | 0–5（0.5 步进） | ✕ |

**风味多维评分**（雷达图维度，0–5，0.5 步进）：
酸 Acidity、甜 Sweetness、醇厚 Body、余韵 Aftertaste、平衡 Balance。外加 **总体 Overall**。

#### PourStage（注水段）
| 字段 | 说明 |
|---|---|
| 段序号 | 1,2,3… |
| 注水至 / 注水量 | 累计水量 or 本段水量（二选一展示，内部统一存累计目标） |
| 目标时间点 | 计划在第几秒（可选） |
| 实际时间点 | 计时器记录的实际落点 |
| 段标签 | 闷蒸 Bloom / 注水 / 断水 等 |

#### RecipeArchive（配方库条目）—— 由"毕业"或"收藏"沉淀
- 来源 Brew 的参数快照 + 豆子的产区/处理法/烘焙度标签。
- 用途：录新豆时按"相似豆"推荐起冲参数。

### 3.2 豆子状态机（Bean.Status）
```
            录入            标记最佳            剩余=0 / 手动毕业
[新建] ──────────▶ [调试中] ──────────▶ [已解锁] ──────────▶ [已毕业(归档)]
                     ▲   │                    │
            未标记最佳 │   │ 取消最佳标记         │ 仍可继续冲(微调/复刻)
                     └───┘                    │
                       继续迭代                 └─▶ 喝完后配方进入【配方库】
```
- **调试中（Dialing）**：已有 ≥1 次冲煮，但还没标记最佳。
- **已解锁（Dialed-in）**：已标记最佳参数。首页高亮"已找到最佳"。
- **已毕业（Archived）**：剩余克数=0 或手动归档；从"在架"移到"已喝完"，最佳配方进入配方库。

> 设计意义：状态机把"调参"变成有目标感、有成就感的进程，这是留存与情绪价值的来源。

## 4. 信息架构与导航

采用 **Tab 三段式**（iOS 26 Liquid Glass TabBar）：

```
┌─ Tab1 豆架 (Shelf)  ← 默认首页
│    在架豆子卡片列表（按状态/养豆期排序）
│    每张卡：养豆天数·消耗进度·最佳分·冲煮次数·状态徽标
│    顶部主操作浮层：⊕ 新建豆子
│    卡片主操作：「冲一杯」「看曲线」
│
├─ Tab2 配方库 (Recipes)  ← 沉淀与复用
│    已毕业/收藏的最佳配方
│    可按 产区/处理法/烘焙度 筛选
│    详情可「以此为起点冲新豆」
│
└─ Tab3 我的 (Me / Stats)
     总览统计：累计冲煮次数、最爱产区、平均分趋势
     设置：单位(g/℃)、默认粉水比、计时器选项、数据导出、外观
```

**深层页面（push / sheet）**：
- 豆子详情 = **实验曲线页**（核心差异化页面，§5.2）
- 冲煮会话页（计时器 + 分段，全屏 sheet，§5.3）
- 复盘页（冲完弹出，§5.4）
- 豆子录入 / 编辑（sheet）
- 分享卡片（sheet）

**导航主路径（黄金路径）**：
```
豆架 →（点某包）冲一杯 → 冲煮会话(计时/分段) → 复盘(打分/下次调) → 回到该豆实验曲线（看到新点）
```
全程 4 步内完成一次"冲+复盘"。

## 5. 核心功能模块

> 优先级：P0 = MVP 必须；P1 = 第一个版本强烈建议；P2 = 后续迭代。

### 5.1 豆子库存（P0）
- 新建/编辑/删除豆子；豆袋拍照。
- 列表卡片信息密度：**养豆天数**（带"最佳赏味期"提示）、**消耗进度条**、**本包最佳分**、**冲煮次数**、状态徽标。
- 养豆期智能提示：依据烘焙度给出建议赏味窗口（如中浅 7–21 天），用颜色提示"养豆中 / 最佳赏味 / 已过赏味"。
- 排序：默认"该冲了"（进入最佳赏味期且最近没冲的靠前）。

### 5.2 实验曲线 / 复盘器（P0）★核心差异化
豆子详情页即"这包的调参历程"：
- **顶部**：豆子信息 + 当前状态（调试中/已解锁）+ 剩余克数进度。
- **参数演化曲线**：横轴=第几次冲煮，纵轴可切换（总分 / 单维风味 / 某个参数如研磨度、水温）。把"评分随迭代上升"的收敛过程可视化。
- **迭代时间线**：每次冲煮一张迷你卡，显示关键参数 + 一句话小结 + "下次调"。最佳那次**置顶高亮**（金色徽标）。
- **对比视图**：勾选 2–3 次冲煮，并排对比参数差异（diff 高亮变化的字段：研磨度↓、水温↓2℃）。
- 主操作：「**再冲一次**」→ 预填**最佳（或最近）**参数进入冲煮会话，鼓励"在上次基础上微调"。

### 5.3 冲煮会话：计时器 + 分段注水（P0）★高频
- **大计时器**：超大字号、不熄屏（idle timer disabled）、可单手起停。
- **分段注水**：预设模板（如"三段式 闷蒸→主注水→收尾"）或自定义；冲煮时点"下一段"打点，记录每段**实际落点时间**与累计水量。
- **目标提示**：若设了目标时间/水量，到点轻微触感反馈（haptic）。
- 可选**总水量进度环**：随分段推进而填充。
- 结束 → 直接进入复盘页，参数已带入。

### 5.4 复盘打分（P0）
- 多维风味雷达（酸/甜/醇厚/余韵/平衡）+ 总分。
- 一句话小结（提供快捷词：太酸/偏苦/水感/平衡/甜感足…）。
- **下次怎么调**：结构化建议器——选维度（研磨/水温/粉水比/注水）+ 方向（更细/更粗、↑/↓），生成"下次：研磨 -1，水温 -2℃"。这条会在下次"再冲一次"时作为提示浮现。
- 一键「**标记为最佳参数**」。

### 5.5 粉水比计算器（P0，内嵌）
- 录参数时三者联动：输入粉量+比例→算水量；或粉量+水量→显示比例。
- 常用比例快捷：1:15 / 1:16 / 1:17。

### 5.6 克数消耗进度（P0）
- 每次冲煮按粉量自动扣减剩余克数；可手动校准（"称了一下还剩 120g"）。
- 剩余预估：还能冲约 N 杯。
- 归零 → 提示"这包喝完了，要把最佳配方存进配方库吗？"

### 5.7 配方库 / 沉淀复用（P1）
- 毕业豆的最佳配方自动入库；也可手动收藏任意一次冲煮为配方。
- 录新豆时若产区/处理法/烘焙度相似，推荐"参考配方"一键带入作为第 1 次起冲参数。

### 5.8 分享卡片（P1）
- 生成精美卡片：豆名 + 最佳参数（研磨/水温/粉水比/总时间）+ 风味雷达/甜感分 + 一句话。
- 风格与 App 视觉一致（见 §7），含可选水印。导出图片/系统分享。

### 5.9 统计与回顾（P2）
- 累计冲煮、最爱产区/处理法、平均分趋势、"调参成功率"（多少包成功解锁）。

### 5.10 其他（P2）
- iCloud 同步、计时器 Live Activity / 灵动岛、Apple Watch 计时、磨豆机刻度档案（不同机器换算）。

### 5.11 功能优先级总览
| 模块 | 优先级 |
|---|---|
| 豆子库存 CRUD | P0 |
| 实验曲线/复盘器 | P0 |
| 冲煮计时器 + 分段注水 | P0 |
| 复盘打分 + 下次调整 | P0 |
| 粉水比计算 | P0 |
| 克数消耗进度 | P0 |
| 最佳参数标记 | P0 |
| 配方库与复用 | P1 |
| 分享卡片 | P1 |
| 统计回顾 | P2 |
| 同步/Watch/灵动岛 | P2 |

**本次开发范围（里程碑 M1）= 全部 P0 + 分享卡片(P1)**，保证产品故事闭环可演示。

## 6. 差异化设计要点（护城河）
1. **以"包"为项目的收敛叙事**：状态机 + 实验曲线，把调参变成有目标的进程。
2. **"下次怎么调" → "再冲一次预填"** 的迭代回路：让上一杯的反思直接驱动下一杯，这是日记类做不到的闭环。
3. **对比 diff 视图**：两次冲煮并排，高亮变了什么参数、风味怎么变——这是玩家最想要的"控制变量法"。
4. **配方库的跨包复用**：把单包经验沉淀为可迁移资产。

## 7. 视觉与设计语言（独特风格 + Apple HIG + Liquid Glass）

### 7.1 设计概念：「实验室手记 × 精品咖啡」
气质 = **温暖的纸感底子 + 实验记录的克制理性 + 萃取过程的液体光泽**。
既要有咖啡的温度和手作感，又要有"调参/数据"的精密感。

### 7.2 色彩
- **底色**：暖白/米纸（非纯白），降低冷感。
- **主色**：咖啡棕 / 浓缩深褐（Espresso）。
- **强调色**：焦糖琥珀（Caramel/Amber）作为"萃取/进度/高亮"色；最佳参数用**金色（Gold）**。
- **语义色**：养豆中（青）/ 最佳赏味（绿）/ 过期（灰褐）；酸（亮黄绿）甜（琥珀）苦（深褐）等风味映射。
- 深色模式：以深咖啡底 + 暖光强调，保持"夜里冲一杯"的氛围。

### 7.3 Liquid Glass（iOS 26）的克制运用
- 浮层控件（TabBar、底部主操作、冲煮会话的悬浮按钮、分享卡片工具栏）使用 `.glassEffect()` / `.buttonStyle(.glass)`，营造"液体光泽"呼应萃取主题。
- 用 `GlassEffectContainer` 管理一组玻璃控件的融合，避免堆叠脏乱。
- **内容区不滥用玻璃**：正文/数据卡片用实体材质卡片，保证可读性（遵循 HIG：玻璃用于导航与控制层，不用于大面积内容）。
- 计时器圆环 / 进度环可借 Liquid Glass 高光做"液面"质感。

### 7.4 标志性视觉元素（让人一眼记住）
- **萃取环（Extraction Ring）**：贯穿计时器进度、消耗进度、分享卡片的统一圆环语言，像滴滤的水线。
- **收敛曲线**：实验曲线页用一条向上收敛的曲线作为页面主视觉与情绪锚点。
- **手写感小结**：一句话小结/标签用偏手记的字体处理，呼应"手记"。

### 7.5 Apple HIG 合规底线
- 系统字体 SF（中文 PingFang）、Dynamic Type 全支持、深浅色、SF Symbols 图标体系。
- 触控目标 ≥44pt；安全区/灵动岛适配；VoiceOver 标签、对比度达标；触感反馈用 `UIFeedbackGenerator`。
- 减少动态/降低透明度等无障碍开关时，玻璃降级为实体材质。

> 详细的视觉规格（间距/圆角/字阶/动效曲线）在《交互设计方案》中给出，并在开发完成后做《UI 视觉审计》。

## 8. 技术架构（实现取向）
- **平台**：iOS 26+（利用 Liquid Glass、最新 SwiftUI）。
- **UI**：SwiftUI（纯声明式，全程 Dynamic Type / 深色 / 无障碍）。
- **持久化**：**SwiftData**（本地优先，单用户，零后端，契合隐私与高频打开）。后续可平滑接 CloudKit 同步。
- **架构**：轻量 MV（SwiftUI View + @Model + 少量 Service：计时器服务、克数结算、养豆期/赏味计算、分享卡片渲染）。
- **无账号、无网络依赖**：纯离线可用，降低使用门槛与隐私顾虑。
- **分享卡片**：`ImageRenderer` 将 SwiftUI 视图渲染为图片。
- **测试**：在 iPhone 17 / 26.5 模拟器上做功能验证（黄金路径 + 边界用例）。

## 9. 风险与开放问题（带入 FRD 进一步收敛）
1. 研磨度无统一单位（每台磨豆机刻度不同）——MVP 用"自由数值 + 记住上次"，不做跨机换算（放 P2）。
2. 剩余克数自动扣减 vs 用户实际损耗（撒粉/残粉）——提供手动校准，避免数字"不可信"。
3. "下次怎么调"结构化 vs 自由文本的平衡——结构化建议 + 允许自由补充。
4. 分段注水的"累计水量"与"本段水量"两种心智——需在交互上明确切换，避免记错。
5. 计时器中途退出/来电/锁屏的状态恢复——需定义会话持久化策略。
