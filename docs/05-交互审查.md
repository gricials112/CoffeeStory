## 1. 审查发现与修正

| ID | 严重度 | 问题（不合理点） | 修正结论（影响界面） |
|---|---|---|---|
| **I-01** | **高** | 准备(sheet) → 会话(fullScreenCover) → 复盘 多层弹层嵌套，转场割裂，且 ActiveBrewSession 生命周期分散在多个 presentation。 | **合并为单一「冲煮流程」全屏覆盖（BrewFlow fullScreenCover）**，内部三阶段：`准备 → 计时 → 复盘`，用顶部进度点/横滑或按钮推进。会话模型只在这一个容器内存活，退出=放弃确认。详 §2.1。 |

| **I-03** | 中 | 曲线纵轴有 9 个选项塞进 segmented control，过载且小屏溢出。 | 纵轴选择改 **横向可滚动 chips，分组**：风味〔总分·酸·甜·醇厚·余韵·平衡〕／参数〔研磨·水温·粉水比〕。默认轴见 I-08。详 §2.3。 |
| **I-04** | **高** | "再冲一次"预填用"最佳/最近"二义；且"下次怎么调"提示可能与预填来源不一致（最近一次是失败试验时）。 | **按状态决定**：`调试中`→预填**最近一次**参数 + 显示其 nextTweak（你在继续实验）；`已解锁`→预填**最佳**参数、不显示纠偏提示（你在复刻），但提供「从最近一次开始」快捷切换。提示与预填来源**始终一致**。详 §2.4。 |
| **I-05** | 中 | 萃取环填充依据"目标时间或目标水量"二义，用户不知道环代表什么。 | **环=累计注水进度（已注水/计划总水量）**，环内大数=已用时（开放、无需目标）。当前段用更亮弧叠加。时间不绑环，避免"没设目标时间环就不动"。详 §2.5。 |
| **I-06** | 中 | 首页全局「冲一杯」多包时强制弹"选哪包"，给最高频操作加了一步。 | 全局主操作默认指向**最近冲过的在架豆**，按钮副标显示该豆名；长按/点豆名处可切换。仅 0 包时该按钮变「录入第一包」。用 `safeAreaInset(.bottom)` 置于 TabBar 之上避免与玻璃 TabBar 重叠。详 §2.6。 |
| **I-07** | 中 | 详情底部浮层塞了 再冲一次/对比/分享 三个，主次不清且与 TabBar 区域冲突（详情已 push 无 TabBar，但仍拥挤）。 | 底部只留 **「再冲一次」prominent** 一个主操作；**对比**移到时间线区的"对比"小按钮（进入多选模式）；**分享**移到导航栏工具栏[G] + 每条冲煮 ⋯ 菜单。详 §2.7。 |
| **I-08** | 低 | 曲线默认轴=总分，但多数试验可能未评分→默认空。 | 默认轴：**有任意评分→总分；全无评分→研磨度**（恒有值，避免空图）。 |
| **I-09** | 中 | "累计/本段"切换若在准备页与会话页各自独立，易记错。 | 升级为**用户偏好（默认累计），全局记忆**，准备页与会话页共享同一显示模式；切换处加一行说明文案。 |
| **I-10** | 低 | 校准剩余靠"点大环"，可发现性差。 | 环旁显式「校准」文字按钮；点环也可，二者并存。 |
| **I-11** | 中 | 复盘"下次怎么调"组合（维度×方向）可能产生无意义组合（如"粉水比 更细"）。 | **方向集合随维度联动**：研磨→更细/更粗；水温/粉水比/粉量→调高/调低（粉水比"调高=更稀"附注）；注水→更快/更慢/多一段/少一段。详 §2.8。 |
| **I-12** | 低 | 会话页"放弃"入口是顶部关闭 X，易与"结束"混淆导致误丢数据。 | 顶部关闭=「退出」二次确认明确写"将不保存这次冲煮"；正常完成只走底部「结束」。两者文案/位置区隔。 |
| **I-13** | 低 | 时间线多选"对比"与每卡 ⋯ 菜单的"对比选中"两套入口。 | 统一为**一个"对比"按钮进入多选模式**（卡片出现勾选圈），选 2–3 后底部"开始对比"。移除 ⋯ 里的对比项。 |
| **I-14** | 低 | 标记最佳的反馈与状态变化在多页发生，用户可能没感知到"解锁"。 | 标记最佳→`notification.success` + 详情头状态徽标变 gold「已解锁」并播一次萃取环金色扫过动效（降低动态时取消）。 |

## 2. 修正后的关键界面规格（开发蓝本）

### 2.1 单一「冲煮流程」容器（I-01）
`BrewFlowView`（`fullScreenCover`），内部 `phase: .prepare / .brewing / .review`，由一个 `@Observable BrewFlowModel` 持有 `ActiveBrewSession`：
- 顶部：豆名 · "第 N 次" · 三点进度指示（准备/计时/复盘）；左上「退出」(I-12)。
- `.prepare`：参数 + 模板（§3.3 内容）。主按钮「开始计时」→ `.brewing`。
- `.brewing`：计时 + 分段（§3.4 内容）。主按钮「结束」→ `.review`；可「上一步」回准备（计时未开始时）。
- `.review`：打分 + 小结 + 下次调（§3.5 内容）。主按钮「保存」→ 提交 Brew、dismiss、回曲线。
- 退出/被杀：`ActiveBrewSession` 持久化，重启可恢复到对应 phase。
- 历史 Brew 的"编辑复盘"走**独立 sheet**（仅 review 形态，无 session、无计时），与本流程区分。

### 2.2 复盘评分输入（I-02）
- 五维：每维一行 = 维度名(色点) + 分段/连续步进控件(0–5,0.5) + 当前值。
- 右上/中央：**只读雷达**实时反映五维；总分单独大数步进。
- VoiceOver：每行 `adjustable`，上下滑加减 0.5。

### 2.3 曲线轴选择（I-03 / I-08）
- 轴 chips 横向滚动，两组以细分隔；选中 chip = amber 实底。
- 默认轴逻辑见 I-08。
- 点折线上的点 → 高亮 + 滚动定位到对应时间线卡。

### 2.4 再冲一次预填与提示（I-04）
```
if bean.hasBest && bean.status == .dialedIn:
    预填 = best.params;  顶部不显示纠偏提示
    提供次按钮「从最近一次开始」→ 切换为 latest.params + latest.nextTweak 提示
else (调试中):
    预填 = latest.params;  顶部显示 latest.nextTweak 提示条
```
准备页顶部提示条仅在"提示存在"时出现。

### 2.5 萃取环语义（I-05）
- 进度 = `累计已注水 / 计划总水量`（0–1）。未设分段计划时，用"最后一段累计=计划总水量"或回退为不显示进度只显示秒。
- 环内：已用时大数（monospacedDigit）。
- 当前段：在环上叠加更亮的一段弧（本段水量占比），帮助"注到哪了"。

### 2.6 首页主操作（I-06）
- `safeAreaInset(.bottom)` 放「冲一杯 · {最近在架豆名}」prominent 玻璃按钮。
- 0 包 → 文案「录入第一包豆子」。
- 点击：直达该豆 BrewFlow（.prepare）。豆名处可点 → 弹豆子选择。

### 2.7 详情底部与工具栏（I-07）
- 底部 `safeAreaInset`：单按钮「再冲一次」prominent[G]。
- 导航栏工具栏[G]：分享、编辑豆子。
- 时间线区头部："对比"按钮（I-13 多选模式）。

### 2.8 下次怎么调·维度×方向联动（I-11）
| 维度 | 可选方向 |
|---|---|
| 研磨 | 更细 / 更粗 |
| 水温 | 调高 / 调低 |
| 粉水比 | 调高(更稀) / 调低(更浓) |
| 粉量 | 增 / 减 |
| 注水节奏 | 更快 / 更慢 / 多一段 / 少一段 |
生成标签如「研磨·更细」「水温·调低」，可多条；附自由备注。

## 3. 复核：黄金路径在修正后的步数
```
豆架「冲一杯·{豆名}」
  → BrewFlow.准备（参数已预填，含上次提示）「开始计时」
  → BrewFlow.计时（下一段×N）「结束」
  → BrewFlow.复盘（步进打分/快捷词/下次调，可选）「保存」
  → 回到该豆实验曲线（新点高亮）
```
核心 4 步、单一容器无层叠跳转；高频"每天冲一杯"路径顺滑。**通过**。
