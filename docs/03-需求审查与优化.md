## 2. 增补规范（v1.1，权威）

### 2.1 研磨度（R-01 修正）
- `Brew.grind: Double`（必填，范围 0–100，0.1 步进，记住该包上次值）。
- `Bean.grinderNote: String?`（可选，如"司令官 C40 / 24 格"），仅作展示与上下文，不参与计算。
- 曲线"研磨度"纵轴使用该数值。跨磨豆机换算明确为 **P2**，M1 不做。

### 2.2 冲煮会话生命周期（R-02 修正）★关键
**模型**：`ActiveBrewSession`（持久化，至多一条；可视为"正在冲的草稿"）
```
beanID, 参数快照(grind,dose,water,ratio,temp),
template(分段模板) + 已记录的 pours[],
timerState: { startedAt: Date?, accumulatedPaused: TimeInterval, isPaused, isRunning, lastPausedAt: Date? },
createdAt
```
**规则**：
1. 点「开始」创建（或覆盖）唯一 session；**此时不创建 Brew、不扣克数**。
2. 计时用墙钟：`elapsed = (now − startedAt) − accumulatedPaused`（暂停时冻结）。被杀重进 App 后据此恢复。
3. 「结束」→ 进入复盘；复盘「保存」时**才** `INSERT Brew` 并 `剩余 -= dose`，随后删除 session。
4. 「放弃本次」二次确认 → 删除 session，不产生 Brew、不扣克数。
5. App 启动若检测到 session 存在 → 顶部恢复条"继续上次冲煮？"（恢复/放弃）。
- **AC-Session-Resume**：计时中冷杀 App，重启后 elapsed 与真实墙钟差 < 1s；放弃后 Brew 数与剩余克数均不变。

### 2.3 分段水量（R-04 修正）
- `PourStage.targetWaterCumulative: Double`（内部唯一真相，单调不减）。
- UI 顶部开关「累计 / 本段」：
  - 累计模式：直接输入/显示累计值。
  - 本段模式：显示 `本段 = 本段累计 − 上一段累计`；输入本段值时回写累计。
- `actualAt: TimeInterval?` 由"下一段/结束"打点写入。
- **AC-Pour-Mono**：任意编辑后累计序列单调不减；切换模式数值自洽。

### 2.4 曲线缺值画法（R-05 修正）
- 纵轴分两类：**评分类**（总分/酸/甜/醇厚/余韵/平衡，可缺）与 **参数类**（研磨/水温/粉水比/总时间，恒有）。
- 评分类：缺值点 = 贴 x 轴的空心点，且与相邻点**断线**（不插值欺骗）。Tooltip 标"未评分"。
- 参数类：全连线。
- 横轴：第 n 次（n 从 1，按时间升序）。
- **AC-Curve-Gap**：含未评分点的总分曲线，缺值处断开且点为空心。

### 2.5 下次怎么调流转（R-06 修正）
- `Brew.nextTweak`: 结构化 `[ {dimension, direction, deltaText?} ]` + `nextTweakNote: String?`。
- 维度枚举：研磨、水温、粉水比、粉量、注水节奏。方向：更细/更粗、调高/调低、增/减。
- **准备下次冲煮**时：读取该包**最近一次 Brew 的 nextTweak**，在准备页顶部以提示条呈现（"上次说：研磨调细一点 · 水温 −2℃"）；用户据此手动改参数（M1 不自动改数值，避免误判）。
- **AC-Tweak-Hint**：A 次复盘写下"研磨更细"，B 次准备页顶部出现该提示。

### 2.6 首启与空态（R-07 修正）
- 首启 3 屏（可跳过、底部页点）：① 这不是日记，是调参实验室 ② 每包豆子一条收敛曲线 ③ 冲完打分→下次怎么调。
- 完成/跳过后落在豆架空态：插画 + 「录入第一包豆子」CTA + 一句话"先把手头这包加进来"。
- 仅首启展示，存 `hasOnboarded` 标志。

### 2.7 内置分段模板（R-08 修正）
以 15g 粉 / 1:16（240g 水）为基准模板（随实际粉水比按比例缩放累计水量）：
| 模板 | 段 | 累计水量(基准240g) | 目标时间 |
|---|---|---|---|
| 单段 | 注水 | 240 | — |
| 闷蒸+单段 | 闷蒸 / 注水 | 30 / 240 | 0:00 / 0:30 |
| **三段(默认)** | 闷蒸 / 注水1 / 注水2 | 30 / 150 / 240 | 0:00 / 0:30 / 1:15 |
| 四段 | 闷蒸 / 注1 / 注2 / 注3 | 30 / 120 / 180 / 240 | 0:00 / 0:30 / 1:00 / 1:30 |
- 模板按当前 `water` 等比缩放（闷蒸固定为 `2×dose`，上限不超首段计划）。
- 用户可在准备页增删段、改累计水量与目标时间；自定义可"存为我的模板"（P1）。

### 2.8 "该冲了"排序评分（R-09）
仅对**在架且未毕业**豆子：
```
score = w1·赏味贴合度 + w2·久未冲惩罚 + w3·调试中加成
赏味贴合度: 在最佳窗口=1，养豆中=0.5，过赏味=0.2，无烘焙日期=0.4
久未冲: min(距上次冲煮天数/7, 1)
调试中加成: 状态=调试中 0.3 / 新建 0.5 / 已解锁 0.1
权重 w1=0.5, w2=0.3, w3=0.2，降序
```
其它排序（最近添加/养豆天数/字母）为静态键。**AC**：最佳赏味期且 5 天没冲的豆排在过赏味豆之前。

### 2.9 相似配方匹配（R-10）
```
similarity = 0.5·(处理法相同?1:0) + 0.3·(产区国家相同?1:0) + 0.2·(1 − |烘焙度档差|/4)
≥0.6 视为相似，取最高分配方推荐；并列取最近归档。
```
缺字段的项不参与对应加分。**AC**：同处理法+同国家的历史配方会被优先推荐。

### 2.10 计时无障碍（R-17）
- 计时数字 `accessibilityLabel="已用时"`，`accessibilityValue` 每 5 秒或分段切换时更新一次（节流），避免 VoiceOver 抢读。
- 分段按钮含明确标签"记录第 N 段，当前累计 X 克"。
