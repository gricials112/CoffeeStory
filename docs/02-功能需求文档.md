# 功能需求文档（FRD）· 这包豆子怎么冲（CoffeeStory）

> 版本 v1.0　|　阶段：功能需求（Phase 2）

## 1. 范围（M1 里程碑）
**纳入**：豆子 CRUD、实验曲线/复盘器、冲煮计时器+分段、复盘打分+下次调整、粉水比计算、克数进度、最佳标记、配方库(基础)、分享卡片。
**不纳入（M1）**：iCloud 同步、Apple Watch、灵动岛 Live Activity、跨磨豆机换算、统计大盘（仅留最简总览）。

## 2. 名词与单位约定
| 名词 | 定义 |
|---|---|
| 一包 Bean | 一袋实体咖啡豆，对应一个"调参项目" |
| 一次冲煮 Brew | 一次实验迭代，归属某一包 |
| 养豆天数 | 今天 − 烘焙日期（天，向下取整；当天=0） |
| 赏味窗口 | 依烘焙度推算的建议饮用区间（见 §4.2） |
| 粉水比 Ratio | 水量 ÷ 粉量，展示为 `1:N`（N 保留 1 位小数） |
| 剩余克数 | 该包当前估计剩余生豆克重 |
| 最佳 isBest | 每包至多一次冲煮被标记为最佳 |

- 重量单位：克（g），整数或 0.1 步进；M1 仅支持公制（设置里预留单位开关，P2）。
- 温度单位：摄氏度（℃）；设置预留 ℉（P2）。
- 时间：秒；展示 `m:ss`。

## 3. 用户角色
单一角色：**玩家（本机用户）**。无多账户、无协作。所有数据本地存储。

## 4. 数据规则（核心，先于用例确定）

### 4.1 字段校验
| 字段 | 规则 | 错误处理 |
|---|---|---|
| 豆名 | 1–40 字，去首尾空格，非空 | 空则禁用保存，提示"给这包起个名" |
| 净含量 | 1–5000 g，整数 | 越界拦截；默认值 200 |
| 烘焙日期 | ≤ 今天；可空 | 选未来日期则拦截 |
| 粉量 Dose | 1–200 g，0.1 步进 | 越界拦截；默认记住上次/15 |
| 水量 Water | 1–3000 g | 与粉量联动校验（见 4.3） |
| 水温 | 60–100 ℃，可空 | 越界拦截；默认 92 |
| 研磨度 Grind | **数值 Double**，0–100，0.1 步进，必填，记住该包上次值（另设 `Bean.grinderNote` 文本说明磨豆机/刻度，不参与计算）— 见 03 文档 §2.1 | 记住该包上次值 |
| 风味各维 | 0–5，0.5 步进 | — |
| 总分 | 0–5，0.5 步进，可空 | — |

### 4.2 赏味窗口推算（依烘焙度）
| 烘焙度 | 建议起始(养豆) | 最佳窗口 | 过赏味判定 |
|---|---|---|---|
| 浅 | 7 天 | 7–28 天 | >35 天 |
| 中浅 | 5 天 | 5–24 天 | >32 天 |
| 中 | 4 天 | 4–21 天 | >30 天 |
| 中深 | 3 天 | 3–18 天 | >28 天 |
| 深 | 2 天 | 2–14 天 | >24 天 |
- 烘焙日期缺失 → 不显示养豆天数与窗口，仅显示"未记录烘焙日期"。
- 数值可调（设置里预留），M1 用上表硬编码。
- **AC-4.2**：给定烘焙度+烘焙日期，列表卡片显示正确的养豆天数与状态色（养豆中/最佳/过赏味）。

### 4.3 粉水比三元联动（计算器）
三个量：粉量 D、水量 W、比例 R（R = W/D）。
- 编辑 **D**：保持 R 不变 → 重算 W = D×R。
- 编辑 **W**：保持 D 不变 → 重算 R = W/D 并刷新显示。
- 编辑 **R**（点快捷 1:15/1:16/1:17 或手输）：保持 D 不变 → 重算 W = D×R。
- 初始：D=上次该包粉量（无则 15），R=设置默认比（默认 1:16），W=D×R。
- **AC-4.3**：任意单字段编辑后，三者恒满足 W = D×R（允许 ±0.1 取整误差）。

### 4.4 剩余克数结算
> 扣减时机：发生在**复盘保存、Brew 正式提交时**，而非冲煮会话开始时（见 03 文档 §2.2 会话生命周期）。放弃会话不扣减。
- 每保存一次冲煮：`剩余 -= 本次粉量`（不小于 0）。
- 删除某次冲煮：`剩余 += 该次粉量`（不超过净含量）。
- 编辑某次粉量 ΔD：`剩余 -= ΔD`（夹紧 0..净含量）。
- 手动校准：用户直接设定剩余值，覆盖估算。
- 剩余到 0：触发"毕业"建议（见 UC-09）。
- **AC-4.4**：连续 3 次冲煮（粉量 15/16/15）后，剩余 = 净含量 − 46；删除中间那次后剩余 = 净含量 − 30。

### 4.5 最佳标记唯一性
- 标记某次为最佳 → 同包其他"最佳"自动取消。
- 取消最佳 → 该包回到"调试中"（若无其他最佳）。
- **AC-4.5**：同一包任意时刻 isBest 数量 ∈ {0,1}。

### 4.6 状态机判定（派生，不手存）
```
brews==0                         → 新建
brews≥1 且 无最佳                 → 调试中
有最佳                           → 已解锁
archived==true(剩余=0确认/手动)   → 已毕业
```
- 毕业是显式标志（用户确认），不因剩余=0 自动毕业（避免数字误差导致误归档）。
- **AC-4.6**：状态随冲煮数/最佳/归档标志正确流转。

## 5. 用例（Use Cases）

> 每个用例含：主路径、备选/异常、相关 FR、AC。

### UC-01 新建豆子（P0）
**前置**：在豆架页。
**主路径**：
1. 点右上 ⊕ → 打开"新建豆子"sheet。
2. 必填：豆名、净含量（默认 200）。其余可填：烘焙商、产区、处理法（枚举+自定义）、烘焙度、烘焙日期、风味标签、拍照、备注。
3. 若选了产区/处理法/烘焙度且配方库有相似项 → 顶部出现"发现相似配方，作为起冲参数？"（P1，可忽略）。
4. 保存 → 回到豆架，新卡片置顶，剩余=净含量，状态=新建。
**异常**：豆名空 → 保存禁用；净含量越界 → 输入拦截。
**相关 FR**：FR-Bean-Create。**AC**：AC-4.1。

### UC-02 浏览豆架（P0）
**主路径**：进入 App 默认落在豆架。卡片显示：封面/占位、豆名、产区·处理法、**养豆天数+赏味状态色**、消耗进度条+剩余 g、最佳分（无则"调试中 n 次"）、状态徽标。
**排序**：默认"该冲了"（进入最佳赏味期且最近未冲优先）；可切换最近添加/养豆天数/字母。
**空状态**：无豆子时展示引导插画 + "录入第一包豆子"。
**AC-5.2**：≥1 包时列表正确渲染各派生值；点击卡片进入实验曲线页。

### UC-03 开始冲煮（黄金路径起点）（P0）
**入口**：豆架卡片「冲一杯」/ 实验曲线页「再冲一次」/ 详情主操作。
**主路径**：
1. 打开"冲煮准备"——预填参数：研磨度=该包上次值；粉量=上次；比例=默认；水温=上次/92；分段模板=上次/默认三段。
   - 若来自「再冲一次」且该包有最佳/最近 → 预填**最佳**参数，并在顶部显示上次的"下次怎么调"提示（如"建议研磨 -1"）。
2. 确认/调整参数（粉水比计算器联动）。
3. 点「开始」→ 进入冲煮会话（计时）。
**异常**：剩余克数 < 粉量 → 黄色提示"这包可能不够这次的粉量"，不拦截（允许，用户可能补豆/不在意）。
**相关 FR**：FR-Brew-Prepare。**AC**：AC-4.3、参数预填正确。

### UC-04 冲煮计时 + 分段注水（P0，高频核心）
**主路径**：
1. 进入会话：大计时器 00:00，屏幕常亮。主按钮「开始计时」。
2. 开始后：计时走动；显示当前段（如"闷蒸"）、目标（若有）。
3. 点「下一段」：记录当前段实际落点（秒）与累计水量；进入下一段；可选 haptic。
4. 总水量进度环随累计水量填充。
5. 点「结束」：停表，记录总时间 → 进入复盘页（UC-05），参数+分段已带入。
**备选**：
- 无模板/不想分段：只用计时器，结束即可（分段为空）。
- 暂停/继续：长计时支持暂停。
- 中途返回/锁屏/来电：**会话状态持久化**，回到 App 可恢复计时（按墙钟时间补偿）或提示"恢复上次冲煮？"。
- 放弃：「放弃本次」二次确认 → 不产生 Brew，剩余克数不扣减。
**相关 FR**：FR-Timer、FR-Pour。**AC**：
- AC-4.7：分段落点严格递增；累计水量单调不减。
- AC-Timer-Resume：杀进程前正在计时，重进 App 能恢复或明确提示。

### UC-05 复盘打分 + 下次怎么调（P0）
**主路径**：
1. 冲煮结束自动进入复盘页，顶部回显本次参数摘要（研磨/粉水比/水温/总时间）。
2. 打分：风味五维雷达（拖拽/步进）+ 总分；可跳过（留空）。
3. 一句话小结：输入框 + 快捷词 chips（太酸/偏苦/水感/平衡/甜感足/尾韵干净…）。
4. "下次怎么调"：选维度（研磨/水温/粉水比/注水/粉量）+ 方向（更细/更粗、+/−、↑/↓）→ 生成结构化建议；可加自由备注。
5. 「保存」→ 写入 Brew、扣减剩余克数、回到实验曲线页（新点出现并高亮）。
6. 可在此直接「标记为最佳」。
**备选**：全部跳过也能保存（至少有参数）。
**相关 FR**：FR-Review。**AC**：AC-4.4（克数扣减）、保存后曲线新增一点。

### UC-06 查看实验曲线 / 复盘这包（P0，核心差异化）
**主路径**：
1. 进入豆子详情（实验曲线页）。
2. 顶部：豆子信息卡 + 状态徽标 + 剩余进度 + 养豆天数。
3. 曲线区：横轴=第 n 次；纵轴可切换（总分/酸/甜/醇厚/余韵/平衡/研磨度/水温/粉水比）。最佳点金色高亮。
4. 时间线：每次冲煮迷你卡（时间、关键参数、小结、下次调）；最佳置顶。
5. 操作：「再冲一次」（→UC-03 预填最佳）、「对比」（→UC-07）、编辑/删除某次。
**AC**：纵轴切换即时重绘；最佳点视觉突出；空数据（0 次）显示"还没冲过，来第一杯"。

### UC-07 对比两次冲煮（P1，强烈建议进 M1）
**主路径**：在时间线勾选 2–3 次 → 进入对比视图，按参数行并排；**变化的字段高亮**并标方向（研磨 22→23、水温 92→90℃↓）；风味雷达叠加。
**AC**：差异字段被正确标注；非差异字段弱化。

### UC-08 标记 / 取消最佳参数（P0）
**入口**：复盘页、时间线迷你卡、对比视图。
**主路径**：点「设为最佳」→ 该包状态→已解锁；旧最佳自动取消；首页卡片显示最佳分+"已解锁"徽标。取消则回退（UC-4.5/4.6）。

### UC-09 克数耗尽 / 毕业归档（P0）
**触发**：保存冲煮后剩余=0，或用户在详情点「这包喝完了」。
**主路径**：弹确认"这包毕业啦🎓 把最佳配方存进配方库？"→ 确认后：Bean.archived=true，移入"已喝完"分组；若有最佳 → 生成 RecipeArchive。
**备选**：无最佳时提示"还没标记最佳，仍要毕业吗？"（允许）。
**AC**：归档后豆架默认隐藏该豆（在"已喝完"可见）；配方库出现该配方（若有最佳）。

### UC-10 手动校准剩余克数（P0）
**入口**：详情页剩余进度条旁「校准」。
**主路径**：输入实际剩余 g → 覆盖估算值。**AC**：AC-4.4 之外，校准后以校准值为基准继续扣减。

### UC-11 编辑 / 删除豆子（P0）
- 编辑：改任意字段；改净含量时重算消耗进度（剩余不自动改，提示可校准）。
- 删除：二次确认，连带删除其下所有冲煮（说明影响）。归档豆也可彻底删除。

### UC-12 编辑 / 删除某次冲煮（P0）
- 编辑参数 → 重算剩余克数（4.4）、刷新曲线。
- 删除最佳那次 → 该包回"调试中"（若无其他最佳）。

### UC-13 配方库浏览与复用（P1）
**主路径**：配方库 Tab 列出已归档/收藏配方；可按产区/处理法/烘焙度筛选；详情显示完整参数 + 来源豆；「以此冲新豆」→ 进入 UC-01 预填这些标签或直接 UC-03 预填参数。
**AC**：筛选正确；复用后参数正确带入。

### UC-14 生成分享卡片（P1）
**入口**：实验曲线页/最佳冲煮「分享」。
**主路径**：渲染卡片（豆名+最佳参数+风味+一句话+萃取环），可选隐藏豆名/加水印 → 系统分享或存图。
**AC**：卡片图片清晰（@3x），暗色/亮色版式正确，无溢出。

### UC-15 设置（P0 最简）
单位（预留）、默认粉水比、计时器(常亮/haptic/默认分段模板)、外观(浅/深/跟随)、数据导出(JSON/CSV，P1)、关于。

## 7. 非功能需求（NFR）
- **性能**：冷启动→豆架 ≤ 1.5s；列表 200 包/单包 200 次冲煮仍流畅滚动。
- **可靠**：冲煮会话崩溃/被杀后可恢复；写操作原子，不产生半条数据。
- **隐私**：纯本地，无网络上报；相册仅在拍照/导出时申请权限。
- **无障碍**：全 Dynamic Type；VoiceOver 完整朗读；对比度 WCAG AA；"降低透明度"时玻璃→实体。
- **本地化**：M1 中文为主，文案集中管理便于后续多语言；数字/日期跟随系统区域。
- **健壮输入**：所有数值字段做范围夹紧与非法输入过滤。

## 8. 边界与异常清单（供 P0/P1 审查重点 review）
1. 烘焙日期为空 → 不显示养豆/赏味，给录入引导。
2. 剩余克数为负的防护（结算夹紧 0）。
3. 计时器超长（>30 min）显示与存储格式。
4. 分段水量回退（用户点错"下一段"）→ 提供撤销上一段。
5. 删除被标记最佳的冲煮 → 状态回退 + 配方库（若已归档生成）如何处理。
6. 同名豆子允许（用昵称/烘焙日期区分）。
7. App 升级数据迁移（SwiftData schema 演进）。
8. 极端 Dynamic Type（AX5）下计时器/曲线/卡片布局不破版。
9. 深色模式下玻璃与金色高亮的对比度。
10. 复盘全跳过（只存参数）→ 曲线该点纵轴值缺失时的画法。
