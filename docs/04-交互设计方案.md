# 交互设计方案 · 这包豆子怎么冲（CoffeeStory）

> 版本 v1.0　|　阶段：交互设计（Phase 4）
> 输入：FRD v1.1（02 + 03）。本文件给出**可直接落地实现**的设计令牌（Design Tokens）、组件规格、逐屏交互与转场、手势/触感/无障碍行为。颜色给十六进制，尺寸给点值（pt）。
> 平台基线：iOS 26，SwiftUI，Liquid Glass。**玻璃只用于导航/控制层，内容卡片用暖色实体材质。**

---

## 1. 设计语言落地：「实验室手记 × 精品咖啡」

三种质感分层（呼应 HIG：内容层 / 导航层）：
- **纸感内容层**：暖白/米纸底 + 实体卡片 + 极克制阴影。承载数据，保证可读性。
- **液体导航层**：Liquid Glass（TabBar、主操作按钮、会话悬浮控件、工具栏）。呼应"萃取/液面"。
- **金色成就层**：最佳参数/解锁状态用金色，稀有、克制，只给"最佳"。

### 2.2 字体（SF / PingFang，全程 Dynamic Type）

| 角色 | 样式 | 备注 |
|---|---|---|
| 页面大标题 | `.largeTitle` | 豆名/页标题 |
| 区标题 | `.title2 / .title3` | 卡片/分区 |
| 强调标题 | `.headline` | 卡片主信息 |
| 正文 | `.body` | |
| 次要 | `.subheadline / .footnote` | 元信息 |
| 标签 | `.caption` | 徽标/单位 |
| **计时数字** | SF Rounded, monospacedDigit, ~64–72pt | 会话页 |
| **参数数值** | SF Rounded + `.monospacedDigit()` | 防跳动 |
| 手记小结 | `.body` italic / serif 备选 | 一句话小结的"手写感"（用斜体近似，避免引入第三方字体） |

字号一律走 `Font.TextStyle` + `.dynamicTypeSize(...)` 上限保护（计时器在 AX 尺寸下封顶，避免破版）。

### 2.3 间距 / 圆角 / 描边 / 动效

- **间距阶**：4 / 8 / 12 / 16 / 20 / 24 / 32（命名 `space.xs…xxl`）。页边距 20，卡片内距 16，卡片间距 12。
- **圆角**：卡片 20，输入/小卡 14，chip 10，玻璃按钮 capsule，进度环按尺寸。
- **描边**：hairline 0.5pt（`line/hairline`）。
- **阴影**：内容卡极轻 `y=2, blur=10, 黑 6%`；金色卡可加暖色微光。
- **动效**：默认 `.spring(response:0.35, dampingFraction:0.85)`；玻璃形变用 `GlassEffectContainer` + `glassEffectID` 的隐式动画；列表/曲线进出用 `.easeInOut(0.25)`。
- **触感**：开始/结束计时 `.impact(.medium)`；下一段 `.impact(.light)`；到达目标 `.impact(.rigid)`；标记最佳/毕业 `.notification(.success)`。

### 2.4 玻璃使用清单（约束）
| 位置 | 处理 |
|---|---|
| TabBar | 系统自动玻璃（Xcode 26 编译即得） |
| 各页工具栏按钮 | 系统玻璃；`.toolbar(removing:.title)` 取得干净玻璃条（视需要） |
| 首页「冲一杯」主操作 | 悬浮 `.buttonStyle(.glassProminent)`，amber tint |
| 会话页悬浮控件（开始/下一段/结束） | `GlassEffectContainer` 包裹，`.buttonStyle(.glass)`/prominent，`glassEffectID` morph |
| 分享卡片工具栏 | 玻璃 |
| **内容卡 / 列表 / 曲线 / 雷达** | **实体材质，禁用玻璃** |
- 无障碍"降低透明度/增强对比"→ 玻璃降级 `.identity`（实体）。

### 2.5 标志性组件：萃取环（Extraction Ring）
统一圆环语言，三处复用：
- **计时进度**：环随目标总时间或目标总水量填充；当前段用更亮的弧。
- **消耗进度**：剩余克数环（豆架卡片小环 + 详情大环）。
- **分享卡**：装饰主图。
规格：描边宽 = 半径×0.12，背景轨 `accent/amberSoft`，进度 `accent/amber`，最佳相关用 `gold/best`。端点圆头。AX：作为 `progressView` 暴露百分比。

## 3. 逐屏交互规格

> 记号：**[G]**=玻璃控件，**[C]**=实体内容卡，转场注明 push/sheet/morph。

### 3.1 豆架（Shelf · Tab1 · 首页）
**布局**：大标题"豆架"+ 顶部右侧工具栏 `⊕ 新建`[G]、排序菜单[G]。下方 `ScrollView` 垂直卡片流。底部悬浮「**冲一杯**」主操作[G, glassProminent]（点后弹"选择今天要冲哪包"或直接对最近一包；见交互②）。`.tabBarMinimizeBehavior(.onScrollDown)`。

**豆子卡片[C]**（高 ~132pt）：
```
┌─────────────────────────────────────────┐
│ [封面48] 豆名(headline)        〔状态徽标〕 │
│          产区·处理法 (subhead, secondary)  │
│  ◷ 养豆 12 天 · 最佳赏味   ●●●○○ 剩 96g    │
│  ★ 最佳 4.5            或  调试中 · 3 次     │
│  [ 冲一杯 ]   [ 看曲线 ]   ← 行内次按钮      │
└─────────────────────────────────────────┘
```
- 养豆天数前圆点用 `state/*` 色；右侧小萃取环=消耗进度。
- 状态徽标：新建(灰)/调试中(amber)/已解锁(gold★)。
- 整卡点击 → 详情(实验曲线) push；"冲一杯"→ 准备页 sheet；"看曲线"=整卡点击同义。
- 左滑：编辑 / 删除（删除二次确认）。
- **空态**：插画 + "录入第一包豆子" CTA（见 §3.9 首启）。
**排序菜单**：该冲了(默认)/最近添加/养豆天数/字母。已毕业豆默认折叠在底部"已喝完(n)"可展开。

**交互②「冲一杯」主操作**：若仅 1 包在架→直接进该包准备页；多包→底部弹出"今天冲哪包？"半屏选择（最近冲过的置顶）。

### 3.2 豆子详情 / 实验曲线（核心页 · push）
**结构**（从上到下，单 `ScrollView`）：
1. **豆子信息头[C]**：封面（可 `.backgroundExtensionEffect()` 让顶部工具栏融入）、豆名大标题、产区/处理法/烘焙度 chips、养豆天数+赏味状态、**大萃取环=剩余克数**（点环→校准 sheet）。状态徽标（已解锁=gold）。
2. **收敛曲线区[C]**：标题"调参历程"。纵轴切换分段控件：总分｜酸｜甜｜醇厚｜余韵｜平衡｜研磨｜水温｜粉水比。折线图：x=第n次，最佳点 gold 实心放大，普通点 amber，未评分点空心断线（R-05）。可点点→高亮对应时间线卡并滚动定位。
3. **迭代时间线[C]**：每次冲煮迷你卡（时间相对显示"3天前"、关键参数行、一句话小结、"下次：…"提示）。最佳卡置顶 + gold 描边 + ★。每卡右上"⋯"：设为最佳/对比选中/编辑/删除/分享。
4. 底部悬浮[G]：「**再冲一次**」prominent（预填最佳/最近 + 顶部带上次 nextTweak 提示）｜「对比」（进入多选）｜「分享」。
**空态（0 次冲煮）**：曲线区显示"还没有数据，来冲第一杯吧"，主操作变「冲第一杯」。
**转场**：从豆架卡片 push；hero 可做封面共享元素（可选）。

### 3.3 冲煮准备（Prepare · sheet, .large）
**目的**：定参数 + 选模板，再开始计时。
**布局**：
- 顶部（若来自再冲一次）**提示条[C, amberSoft]**：「上次说：研磨调细一点 · 水温 −2℃」。
- **参数组[C]**：研磨度（数值 stepper + 大显示）、**粉水比计算器**（粉量 / 水量 / 比例 三块联动，1:15/1:16/1:17 快捷 chips）、水温（stepper，默认带入）。数值用 Rounded monospaced。
- **分段模板[C]**：分段选择（单段/闷蒸+单段/三段默认/四段/自定义）。展开可编辑每段累计水量+目标时间；顶部「累计/本段」切换开关（R-04）。
- 剩余不足提示：`剩余 < 粉量` → 黄条非阻断提示。
- 底部[G]「开始」glassProminent。
**校验**：必填(研磨/粉量/水量) 缺失禁用开始。
**转场**：sheet 上滑；点开始→**全屏覆盖**进入会话（非 dismiss，避免误触底层）。

### 3.4 冲煮会话（Brew Session · fullScreenCover）★高频
**设计原则**：站在秤前、单手、看得清、不熄屏。
**布局（垂直居中）**：
```
        〔豆名 · 第4次〕            ← 顶部轻信息
   ┌───────────────────┐
   │     萃取环(大)       │  环内：
   │       1:02         │  大计时(64-72pt monospaced)
   │   闷蒸 → 注水1      │  当前段名 + 目标"→150g / 0:30"
   └───────────────────┘
     累计水量 96 / 240 g     ← 线性副指示
   段落清单（已打点的段，带实际秒）
 ───────────────────────────
 [G 悬浮控件容器 GlassEffectContainer]
   ⟲撤销段   �⏸暂停   ⏭下一段(prominent)   ■结束
```
**交互**：
- 进入即"待开始"，主按钮「开始计时」[G prominent]；按下→`.impact(.medium)`、`isIdleTimerDisabled=true`、环开始走。
- 「下一段」：记录当前段 actualAt + 累计水量，morph 到下一段（`glassEffectID` 过渡），`.impact(.light)`；到达目标时间 `.impact(.rigid)`。
- 「撤销段」：回退上一段打点（R-14）。
- 「暂停/继续」：冻结 elapsed（会话模型 accumulatedPaused）。
- 「结束」：`.impact(.medium)`、停表 → 进入复盘（同一 fullScreenCover 内 push 或替换为复盘）。
- 「放弃」（顶部关闭→二次确认）：丢弃 session、不建 Brew、不扣克数。
- **恢复**：若 App 重启检测到 session，豆架顶部出现恢复条；点"继续"直接回到此页按墙钟续算。
**无障碍**：计时 `accessibilityValue` 每 5s/分段节流播报；按钮全标签化（R-17）。
**Dynamic Type**：计时字号封顶，超大字时副指示折行不破环。

### 3.5 复盘（Review · 接会话之后 / 也可对历史 Brew 编辑）
**布局**（`ScrollView`）：
1. **参数回显[C]**：研磨 / 1:16 / 92℃ / 2:36，只读摘要（可点"展开看分段"）。
2. **风味雷达[C]**：五维（酸/甜/醇厚/余韵/平衡）可拖拽五个滑杆或直接拖雷达顶点；右侧总分大数（0–5，0.5 步进）。各维用 `flavor/*` 色。
3. **一句话小结[C]**：文本框 + 快捷词 chips（太酸/偏苦/水感/平衡/甜感足/尾韵干净/醇厚…）点选追加。
4. **下次怎么调[C]**：维度 chips（研磨/水温/粉水比/粉量/注水）×方向（更细·更粗 / 调高·调低 / 增·减）组合，生成结构标签可多条；附自由备注。
5. 行内开关「设为这包的最佳参数 ★」。
6. 底部[G]「保存」prominent。保存→写 Brew、扣克数、`.notification(.success)`、回到实验曲线（新点高亮滚入）。若剩余=0 → 触发毕业弹窗（§3.6）。
**全跳过**：仅参数也可保存（评分留空，曲线按 R-05 处理）。

### 3.6 毕业 / 归档（确认弹窗 alert/sheet）
剩余=0 或手动"这包喝完了"→ 弹"这包毕业啦 🎓 把最佳配方存进配方库？"。
- 有最佳→「存入并毕业」/「只毕业」/「取消」。
- 无最佳→提示"还没标记最佳，仍要毕业？"。确认→ `archived=true`，移入"已喝完"，生成 RecipeArchive（若有最佳），`.notification(.success)` + 轻庆祝动效（萃取环金色一圈）。

### 3.7 对比（Compare · sheet）
从时间线多选(2–3)进入：
- 顶部分段选中的次序卡缩略。
- **参数对照表[C]**：行=研磨/粉量/粉水比/水温/总时间/各风味；列=所选各次。**变化字段高亮 + 方向箭头**（22→23↑、92→90℃↓）。
- 雷达叠加（≤3 条，不同描边低透明填充，R-13）。
- 可在此直接把某次「设为最佳」。

### 3.8 配方库（Recipes · Tab2）
- 列表[C]：配方卡（来源豆名快照、产区/处理法/烘焙度 chips、核心参数、风味缩略）。
- 顶部筛选[G 工具栏]：产区/处理法/烘焙度。
- 详情：完整参数 + 来源 + 「以此冲新豆」(→ 新建豆子预填标签 / 或对已有豆开准备页预填参数)。
- 空态："还没有沉淀的配方，调出一包的最佳参数后会出现在这里"。

### 3.9 首启引导 + 新建/编辑豆子
- **首启 3 屏**：横滑 PageView，底部页点，"跳过/开始"。文案见 03 §2.6。
- **新建豆子 sheet（.large）**：分组表单——基本(豆名*、净含量*、封面拍照)｜风味来源(产区、处理法 picker+自定义、烘焙度 segmented、烘焙日期、风味标签)｜其它(烘焙商、备注、磨豆机说明)。顶部"相似配方"提示条（P1）。底部「保存」。校验：豆名/净含量。
- **编辑豆子**：同表单预填；改净含量提示可"校准剩余"。

### 3.10 我的 / 设置（Tab3）
- 顶部最简总览[C]：累计冲煮次数、在架/已毕业数、平均分。
- 设置组：默认粉水比、计时器(常亮/触感/默认模板)、外观(浅/深/跟随)、关于。导出占位(禁用，P2)。

## 4. 导航 / 转场总图
```
TabView[G]
 ├ 豆架 (NavigationStack)
 │   └ 详情/曲线 (push)
 │        ├ 准备 (sheet.large) ─▶ 会话 (fullScreenCover) ─▶ 复盘(同cover) ─▶ dismiss回曲线
 │        ├ 对比 (sheet)
 │        ├ 校准剩余 (sheet.medium)
 │        └ 分享 (sheet)
 │   └ 新建/编辑豆子 (sheet.large)
 ├ 配方库 (NavigationStack) └ 配方详情(push)
 └ 我的 (NavigationStack) └ 设置项
首启引导：覆盖于 TabView 之上的 fullScreenCover（hasOnboarded=false 时）
会话恢复条：豆架顶部，hasActiveSession 时
```
转场原则：纵深用 push；任务型（准备/复盘/新建）用 sheet；沉浸型（会话）用 fullScreenCover；玻璃控件用 morph。

## 7. 无障碍验收点（开发自检）
- 全部文本 Dynamic Type；AX5 下各页不破版（计时器/曲线/雷达有封顶与折行策略）。
- VoiceOver：卡片合并朗读"豆名，状态，养豆n天，剩余x克，最佳y分"；计时节流播报；按钮均有 label/hint。
- 对比度 WCAG AA：金色/amber 文字在浅深底均达标（必要时配深描边）。
- 降低透明度/增强对比 → 玻璃 `.identity` 实体降级；降低动态 → 去除 morph/庆祝动效。
- 触控目标 ≥44pt；色彩不作为唯一信息载体（状态同时有文字/图标）。
